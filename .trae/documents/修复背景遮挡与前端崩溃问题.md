# 错误分析报告：背景遮挡与内容渲染失败

经过对代码库和报错信息的深入分析，以下是关于"背景页面遮挡主内容"问题的详细诊断报告。

## 1. 核心问题诊断

**现象描述**：用户看到的"背景遮挡内容"实际上是**前端应用崩溃**导致的。由于 JavaScript 执行出错，主要内容组件（Home）无法渲染，只留下了位于 DOM 结构外层且具有固定定位（fixed）的背景组件（InkCanvas），给用户造成了"背景遮挡"的视觉错觉。

**报错信息**：
- **错误**：`Uncaught TypeError: Cannot read properties of undefined (reading 'length')`
- **位置**：`index-BGGHpLNq.js` (构建后的 Home 页面逻辑)
- **原因**：前端代码尝试读取一个 `undefined` 对象的 `length` 属性。具体发生在 `Home.jsx` 中检查 `posts.length` 时，因为 `posts` 状态变成了 `undefined`。

## 2. 根本原因分析

问题的根源在于**前后端连接配置缺失**，导致 API 请求失败但未被正确处理。

1.  **API 请求异常**：
    - 前端配置的 API 地址是 `/api`（相对路径）。
    - 在开发环境（Vite）中，前端运行在 `localhost:5173`，后端运行在 `localhost:3001`。
    - 由于 `vite.config.js` 中**缺少代理（Proxy）配置**，前端发出的 `/api/posts` 请求被 Vite 开发服务器拦截。
    - Vite 将此请求视为未知的路由，根据 SPA（单页应用）规则，返回了 `index.html` 的内容（状态码 200 OK）。

2.  **数据解析失败**：
    - `frontend/src/lib/api.js` 中的 `request` 函数尝试将响应解析为 JSON：`response.json()`。
    - 由于返回的是 HTML（`<!DOCTYPE html>...`），JSON 解析失败。
    - `catch` 块捕获了错误并返回了空对象 `{}`。

3.  **状态污染与崩溃**：
    - `usePagedPosts.js` 接收到空对象 `{}` 作为 `data`。
    - 代码执行 `setPosts(data.items)`，导致 `posts` 状态被设置为 `undefined`。
    - `Home.jsx` 渲染时执行 `posts.length`，触发 `TypeError`，导致组件树崩溃，只剩下未受影响的背景层。

## 3. 修复方案

### 步骤 1：配置 Vite 代理 (必须)
在 `frontend/vite.config.js` 中添加代理配置，将 `/api` 请求转发到后端服务器。

### 步骤 2：增强错误处理 (建议)
优化 `usePagedPosts.js`，防止因 API 返回格式错误而导致整个应用崩溃。
